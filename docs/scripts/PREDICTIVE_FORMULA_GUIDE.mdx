# Predictive Formula System Guide

A unified system for predicting planetary precession fluctuations for all 7 planets (Mercury through Neptune) based on the Holistic Universe Model. For the theoretical derivation of these formulas (Fibonacci hierarchy, resonance loops, term categories), see [Formula Derivation](../35-formula-derivation.md).

## Overview

This system predicts how planetary precession rates deviate from their baseline values over time. The model uses a **unified 429-term feature matrix** that captures the complex interactions between:

- Earth's perihelion longitude and its rate of change (ERD)
- Planet perihelion longitudes relative to Earth
- Earth's obliquity and eccentricity variations
- Multiple harmonic periods derived from the Holistic Year (H; see [Constants Reference](../20-constants-reference.md))

All 7 planets use the **same feature structure** with planet-specific coefficients trained via ridge regression.

## Key Concepts

### Holistic Year (H)

The fundamental cycle of H years from which all other periods are derived (see [Constants Reference](../20-constants-reference.md) for current value):

| Period | Description |
|--------|-------------|
| H | Master cycle |
| H/3 | Inclination cycle |
| H/8 | Obliquity cycle |
| H/13 | Axial precession |
| H/16 | Earth perihelion |

### Planet Periods

Each planet's precession period is derived from H using Fibonacci-related fractions:

| Planet | Period Formula | Baseline Rate |
|--------|---------------|---------------|
| Mercury | H × 8/11 | +531.9 "/cy |
| Venus | H × 2 | +193.4 "/cy |
| Mars | H × 3/13 | +1676.4 "/cy |
| Jupiter | H / 5 | +1934.3 "/cy |
| Saturn | H / 8 | -3094.9 "/cy |
| Uranus | H / 3 | +1160.6 "/cy |
| Neptune | H × 2 | +193.4 "/cy |

### J2000 Calibration

All planets are calibrated to their official J2000 (year 2000) perihelion longitudes. The perihelion at any year is calculated as:

```
θ(year) = θ₀ + 360° × (year - 2000) / period
```

## Quick Start

### Running Predictions

From the command line:

```bash
cd docs/scripts
python predict_precession.py 2000
```

Output shows predictions for all planets at the specified year.

### Using the Python API

```python
from predict_precession import predict, predict_total, get_all_predictions

# Get fluctuation for a single planet
fluctuation = predict(2000, 'mars')  # Returns -113.93 "/cy

# Get total precession rate (baseline + fluctuation)
total = predict_total(2000, 'mars')  # Returns +1568.07 "/cy

# Get predictions for all planets
results = get_all_predictions(2000)
for planet, data in results.items():
    print(f"{planet}: {data['total']:.2f} \"/cy")
```

### Getting Earth Parameters

```python
from predict_precession import get_earth_parameters

earth = get_earth_parameters(2000)
print(f"Perihelion: {earth['perihelion']:.6f}°")
print(f"Obliquity: {earth['obliquity']:.6f}°")
print(f"Eccentricity: {earth['eccentricity']:.10f}")
print(f"ERD: {earth['erd']:.10f} °/yr")
```

### Getting Planet Obliquity

```python
from predictive_formula import calc_planet_obliquity

# Dynamic obliquity anchored to J2000 axial tilt
for planet in ['Mercury', 'Venus', 'Mars', 'Jupiter', 'Saturn', 'Uranus', 'Neptune']:
    obliq = calc_planet_obliquity(planet, 2000)
    print(f"{planet}: {obliq:.4f}°")
```

Obliquity oscillates with Fibonacci-predicted periods (see [doc 37](../37-planets-precession-cycles.md)). Venus and Neptune return static values (no obliquity cycle).

### Cardinal Point Prediction (Solstices & Equinoxes)

All four cardinal points (SS, WS, VE, AE) are supported via an optional `cp_type` parameter (default `'SS'` for backward compatibility):

```python
from predictive_formula import calc_solstice_ra, calc_solstice_jd, calc_solstice_year_length

# Where: RA of any cardinal point (fully derived, zero fitted constants)
ra_ss = calc_solstice_ra(2000, 'SS')   # 90.0169° (summer solstice ≈ 6h)
ra_ws = calc_solstice_ra(2000, 'WS')   # 270.0169° (winter solstice ≈ 18h)
ra_ve = calc_solstice_ra(2000, 'VE')   # 0.0169° (vernal equinox ≈ 0h)
ra_ae = calc_solstice_ra(2000, 'AE')   # 180.0169° (autumnal equinox ≈ 12h)

# When: Julian Day of any cardinal point (anchored at observed J2000 value)
jd_ss = calc_solstice_jd(2000, 'SS')   # 2451716.575000 (Jun 21, 01:48 UTC)
jd_ws = calc_solstice_jd(2000, 'WS')   # 2451900.067346 (Dec 21, 01:37 UTC)

# How long: time between consecutive events of the same type
yr_ss = calc_solstice_year_length(2000, 'SS')  # 365.24160 days (shortest, −51s)
yr_ws = calc_solstice_year_length(2000, 'WS')  # 365.24274 days (longest, +47s)
```

The RA formula uses only H/3 and H/8 harmonics with amplitude `A/sin(ε)` — a pure geometric projection of the scene graph. The JD formula uses 12 harmonics per cardinal point (5 Fibonacci fundamentals + 7 overtones that are sums of Fibonacci: H/6=3+3, H/11=3+8, H/32=16+16, etc.). The H/16 phase rotates 90° between consecutive cardinal points, encoding the equation of center sweeping through the orbit.

RMSE: RA 0.089° for all types; JD: SS 2.7min, VE 3.0min, AE 5.0min, WS 5.3min over full H. See [doc 14](../14-solstice-prediction.md) for full derivation.

## Retraining the Model

If you have updated observed data, you can retrain the coefficients:

### 1. Verify Earth Perihelion Formula

Before training, verify the Earth perihelion formula produces correct values at J2000:

```python
cd docs/scripts
python -c "from predictive_formula import calc_earth_perihelion; print(f'{calc_earth_perihelion(2000):.6f}°')"
```

Expected: ~102.94° (within ~0.01° of the Excel `Earth Perihelion ICRF` value at year 2000). The training script also prints this value automatically.

### 2. Prepare Training Data

The training data is in `98-holistic-year-objects-data.xlsx` (Excel format). The step size should divide H (335008) evenly — currently 29-year steps (11553 points). Data comes from:

**Sheet 'Holistic_objects_PerihelionPlan':**
- `Model Year` — Calendar year (must include year 2000)
- `{Planet} Precession Fluctuation` — Observed fluctuations (″/century) for Mercury, Venus, Mars, Jupiter, Saturn, Uranus, Neptune

Earth orbital parameters (obliquity, eccentricity, perihelion) are computed analytically by `predictive_formula.py`, not read from Excel.

### 3. Run Training

```bash
cd docs/scripts
python train_precession.py
```

This will:
1. Load observed data from Excel
2. Build the 429-term feature matrix for each data point
3. Fit ridge regression (α=0.01) for each planet
4. Write new coefficient files (`*_coeffs_unified.py`)
5. Display RMSE and R² metrics

### 4. Update JavaScript

After training, copy the new coefficients into `src/script.js` `PREDICT_COEFFS` object. Each planet needs all 429 coefficients updated.

### 5. Validate Results

```bash
python validate_precession.py
```

This compares predictions against observed data and shows:
- RMSE, Mean Error, Max Error for each planet
- R² scores (should be > 0.998 for good fit)
- Sample comparisons at key years

## Feature Matrix Architecture

The 429-term unified feature matrix is organized into 25 groups:

| Group | Terms | Description |
|-------|-------|-------------|
| 1 | 18 | Angle terms (δ, Σ, harmonics) |
| 2 | 6 | Obliquity/Eccentricity terms |
| 3 | 12 | ERD terms (linear, quadratic, cubic) |
| 4 | 22 | Periodic terms (11 periods × 2) |
| 5 | 22 | ERD × Periodic |
| 6 | 12 | ERD² × Periodic |
| 7 | 24 | Periodic × Angle |
| 8 | 16 | ERD × Periodic × Angle |
| 9 | 16 | Periodic × 2δ |
| 10 | 12 | Periodic × Periodic |
| 11 | 1 | Constant |
| 12 | 4 | ERD × Sum-angle |
| 13 | 60 | Extended harmonics (3δ, beats) |
| 14 | 16 | Venus periodic terms |
| 15 | 12 | Time-varying obliq/ecc (critical for Saturn) |
| 16 | 20 | Venus fine-tuning |
| 17 | 14 | Greedy round-1: H/60, H/24×angle, H/45×angle, H/9×2δ |
| 18 | 16 | Greedy round-2: H/110, H/60×angle, H/75, H/41×angle, H/56, H/45 |
| 19 | 16 | Greedy round-3: H/56×angle, H/41×2δ, ERD×H/110, ERD×6δ, H/39×angle |
| 20 | 18 | Greedy round-4: H/60×2δ, H/112×angle, H/39×2δ, H/96×angle, H/37 |
| 21 | 22 | Greedy round-5: H/96×2δ, H/7+angle, H/18, H/37×2δ, H/38×2δ, ERD×6δ |
| 22 | 10 | Greedy round-6: H/31 bundle (Venus dominant, ΔRMSE -1.55) |
| 23 | 20 | Greedy round-7: H/128 bundle, H/42 bundle (Venus) |
| 24 | 18 | Saturn high-frequency harmonics: H/(8×34) harmonic series |
| 25 | 22 | Venus high-frequency triplets: carrier ± 16 sideband structure |

### Saturn Note

Saturn's precession period (H/8) equals the obliquity cycle, creating strong coupling with Earth's obliquity/eccentricity variations. The GROUP 15 terms are critical for Saturn's accuracy.

### Groups 17–25 Note

Groups 17–23 were added via greedy forward feature selection (`greedy_features.py`) over 7 rounds. Each round identifies candidate features with highest residual correlation, refits with ridge regression, and selects the terms providing the largest RMSE reduction. Groups 24–25 add planet-specific high-frequency harmonics: Saturn's H/(8×34) series (GROUP 24) and Venus's carrier ± 16 sideband triplets (GROUP 25).

## Extending the Model

### Adding New Features

To add features to the model:

1. Run `greedy_features.py` to find candidate terms with highest residual correlation
2. Edit `predictive_formula.py`: add period constants, add a new GROUP to `build_features()`, update the docstring term count
3. Retrain all planets with `train_precession.py`

### Adding a New Planet

1. Add planet configuration to `PLANETS` dict in `predictive_formula.py`:
   ```python
   'newplanet': {
       'name': 'NewPlanet',
       'period': H * fraction,  # Derived from H
       'theta0': xxx.xxxxx,     # J2000 perihelion longitude
       'baseline': 1296000 / period * 100,  # or negative for retrograde
   }
   ```

2. Add column to training CSV: `NewPlanet Precession Fluctuation`

3. Retrain to generate `newplanet_coeffs_unified.py`

4. Import in `predict_precession.py`

## Mathematical Background

### Precession Rate

Total precession rate = Baseline + Fluctuation

- **Baseline**: Mean precession rate derived from planet's period (360°/period per year, converted to "/cy)
- **Fluctuation**: Time-varying deviation predicted by the model

### Feature Calculation

Key calculated values for each prediction:

1. **Earth Perihelion** (θ_E): Sum of mean motion + 21 harmonic terms
2. **ERD**: Analytical derivative of Earth perihelion (rate deviation)
3. **Planet Perihelion** (θ_P): Linear from J2000 value
4. **Difference Angle** (δ): θ_E - θ_P (key interaction term)
5. **Obliquity/Eccentricity**: Time-varying Earth parameters

The model captures how these parameters interact across multiple timescales to produce the observed fluctuation patterns.
